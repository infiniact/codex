# Output Formatting

## Headers
- Use only when improving clarity
- Short (1-3 words), `**Title Case**`
- No blank line before first bullet

## Bullets
- Use `-` followed by space
- Merge related points; one line preferred
- 4-6 bullets per group, ordered by importance

## Monospace
- Wrap commands, paths, env vars, code in backticks
- Don't mix monospace and bold

## File References
- Inline code for clickable paths
- Format: `path:line` or `path#Lline`
- No URIs (file://, vscode://)
- Examples: `src/app.ts:42`, `server/index.js#L10`

## Structure
- Related bullets together
- Order: general → specific → supporting
- Match structure to complexity

## Tone
- Collaborative, natural
- Concise, factual, no filler
- Present tense, active voice

## Don'ts
- No nested bullets
- No ANSI codes
- No long keyword lists
